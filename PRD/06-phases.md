# 6. 개발 우선순위 (Phase)

### Phase 1 — 핵심 기반 (MVP) ✅
> 목표: 프로젝트 중심의 기본 업무 흐름 구축

| 모듈 | 기능 | 상태 |
|------|------|------|
| 시스템 | 인증, RBAC, 공통 코드, 레이아웃 | ✅ |
| 영업 | 고객 관리, 수주 등록, 수주→프로젝트+설계공정 자동 생성 | ✅ |
| 프로젝트 | 프로젝트 CRUD, 상태 관리(자동 전환), E2E 타임라인, 자재 KPI | ✅ |
| 생산 | 공정 계획 (설계 공정 포함), 작업 지시, 실적 입력, WO↔Project 상태 동기화 | ✅ |
| 대시보드 | 프로젝트 현황, 작업 현황 | ✅ |

### Phase 2 — 확장 ✅
> 목표: 자재/구매, 외주, BOM 관리 체계화

| 모듈 | 기능 | 상태 |
|------|------|------|
| 설계 | BOM→PR 자동 생성, 설계 공정 자동 시드 | ✅ |
| 자재 | 자재 마스터, 재고 관리, 입출고, 서버 페이지네이션, STEEL 치수 | ✅ |
| 구매 | 구매 요청, 발주, 입고 검수, PR→PO 프로젝트별 자동 분할, project_id 전파 | ✅ |
| 외주 | 외주 발주, 일정 추적, 입고 | 미구현 |
| 견적 | 견적 등록, 원가 산출, 견적서 생성 | 미구현 |

### Phase 3 — 고도화 (일부 완료)
> 목표: 품질·원가·설비 관리 및 분석 역량 강화

| 모듈 | 기능 | 상태 |
|------|------|------|
| 품질 | 검사 관리, 트라이아웃, 불량/클레임 | 미구현 |
| 원가 | 프로젝트 원가 집계, 견적 vs 실적 비교 | 미구현 |
| 설비 | 설비 마스터, 보전 관리, 가동률 | 미구현 |
| 스케줄링 | 간트 차트, 부하 관리, 납기 역산 | 미구현 |
| 회계 | 자동분개 엔진(판매/구매/재고/출고 4종), 전표 조회, 미수/미지급 | ✅ Step 1-2 완료 |
| 데이터 정합성 | project_id 누락률, 상태 불일치, 문서 연결 점검 | ✅ |
| 통계 (기본) | 영업 통계 (매출 추이, 입금 현황), 생산 통계 (납기 준수율, 가동률) | 미구현 |

### Phase 4 — 최적화
> 목표: 데이터 기반 의사결정 및 자동화

| 모듈 | 기능 |
|------|------|
| 대시보드 | 경영 분석 대시보드 고도화 |
| 통계 (고도화) | 품질·원가·자재 통계, 경영 종합 통계, 전년 동기 비교, 월간 리포트 자동 생성 |
| 리포트 | 각종 분석 리포트 자동 생성 (PDF 내보내기) |
| 알림 | 지연 예측 알림, 재고 부족 알림, 미수금 알림 |
| 연동 | 세금계산서 API, CAD 데이터 연동 |
